# Mitozz Asset Set Architecture Recommendation

## Purpose

This document defines how many asset sets Mitozz should actually operate with in order to reduce Nano Banana token waste, improve visual consistency, and keep the account varied without drifting away from the brand system.

The goal is not to create more templates.
The goal is to create fewer, stronger, more reusable template families with controlled variation.

## Strategic Call

For Mitozz's current launch-stage cadence, use:

- `5` master template families
- `8` locked operational sets
- `3` locked product plates inside the product family

Do not expand beyond that until the posting volume or campaign complexity clearly earns it.

## Why This Is The Right Number

If the system is smaller than this:

- the feed risks becoming too repetitive
- the account can become too white-card heavy
- trust, human rhythm, and product moments will feel underpowered

If the system is larger than this:

- prompts will become harder to control
- the model will waste tokens reinventing layout decisions
- typography, spacing, and visual mood will drift more often
- the account will start to feel like multiple brands sharing one feed

For Mitozz's April 2026 plan, the actual need is concentrated in:

- educational carousels
- calm product tie-backs
- reel and lifestyle context
- trust and authority moments
- simplified story support

That mix does not justify a large template library.

## What The External Scan Suggests

The strongest Japanese brand systems reviewed during this pass point to the same operating pattern:

- they keep one umbrella visual world, not many equally weighted aesthetics
- they separate categories, sub-brands, or use cases into recurring series rather than redesigning every post
- they treat collaborations and seasonal moments as capsules layered on top of the base identity
- they repeat a few recognizable content modes so the account stays memorable

Implications for Mitozz:

- do not build a new permanent set for every content topic
- do not treat every slide count as a different design system
- do not let AI interpret fonts, spacing, and layout from scratch on every run

## Recommended Architecture

### Master Family 1: Education Signature

Role:

- primary Mitozz feed identity
- educational covers
- myth-to-clarity posts
- save-worthy science explainers

Locked operational sets:

- `E1` `Education Hook White Card`
- `E2` `Editorial Science Body`

Rule:

- this should remain the most repeated family on the grid

### Master Family 2: Product Presence

Role:

- product familiarity
- end-card CTA moments
- ritual relevance
- premium product reassurance

Locked operational sets:

- `P1` `Cool Product Hero`
- `P2` `Warm Ritual Product`

Locked product plates:

- `Plate A` clean tabletop with cool editorial light
- `Plate B` softer curtain-light atmosphere
- `Plate C` warmer ritual-led setting

Rule:

- do not build many different product worlds
- keep one product family and rotate only the plate logic

### Master Family 3: Lifestyle / Motion Context

Role:

- reel source frames
- daily-condition storytelling
- routine scenes
- bridge slides between education and product

Locked operational sets:

- `L1` `Daily Context Realism`
- `L2` `Motion Hook Still`

Rule:

- these sets should create relief, not dominate the feed

### Master Family 4: Trust / Authority

Role:

- expert reassurance
- premium proof
- founder or advisor-style confidence
- calm human warmth

Locked operational set:

- `T1` `Warm Trust Portrait`

Rule:

- one trust family is enough for now
- variation should come from crop, age range, wardrobe, and environment, not from new layouts

### Master Family 5: Story Utility

Role:

- feed reinforcement
- low-friction education
- route-to-feed or route-to-action Stories

Locked operational set:

- `S1` `Story Utility`

Rule:

- do not create separate evergreen story sets for polls, question boxes, proof cards, and recap slides
- use one simplified system and let native Instagram stickers create the interaction behavior

## Final Count

Use these `8` locked operational sets:

1. `E1` `Education Hook White Card`
2. `E2` `Editorial Science Body`
3. `P1` `Cool Product Hero`
4. `P2` `Warm Ritual Product`
5. `L1` `Daily Context Realism`

Supplement them with these `3` product plates:

1. `Plate A`

This is enough variation to keep the account alive without making production chaotic.

## Family Versus Blueprint Rule

A set should never be treated as a fixed slide count.

A set is:

- a visual family
- a spacing system
- a typography system
- a copy hierarchy
- an image-treatment logic

A blueprint is:

- the number of slides
- the role of each slide
- the sequence of those slide roles

This means:

- if `Set A` was originally shown as `5` slides, it should still be reusable as a `3`-slide or `7`-slide post
- the team should choose only the needed page roles for that post instead of inventing a new set

## Token-Saving Production Rule

The biggest savings will not come from reducing the number of prompts alone.
They will come from reducing how much the model is allowed to invent inside each prompt.

Lock in code, template, or master-layout form:

- font family
- font role and weight
- spacing
- safe areas
- text hierarchy
- card radius
- border treatment
- CTA structure

Allow AI to vary:

- image plate
- camera distance
- crop
- background softness
- lighting warmth
- lifestyle setting
- portrait subject
- abstract science cue

Do not ask AI to decide:

- fonts
- spacing systems
- line-break style
- exact text hierarchy
- slide-level layout structure

That is where most drift and token waste are happening.

## Variant Rule

Only high-frequency content modes should earn multiple variants.

Use this threshold:

- if a mode appears `4` or more times per month, it can earn a second locked variant
- if a mode appears `8` or more times per month, it can earn a third controlled plate or crop variant
- if a mode appears less than `4` times per month, keep it inside one family

For Mitozz right now:

- education deserves `2` locked sets
- product deserves `2` locked sets plus `3` plates
- reel/lifestyle deserves `2` locked sets
- trust needs `1` locked set
- stories need `1` locked set

## Refresh Cadence

Do not redesign the library monthly.

Use:

- minor additions every `6` to `8` weeks only if repetition becomes obvious
- major refresh every quarter
- full redesign only if strategy or brand direction changes

## How To Use The Locked-In Designs The User Will Provide

When the base references arrive, use this workflow:

1. map each provided design to one of the `8` locked set slots above
2. mark what is immutable versus what can vary
3. split every set into page-role modules, not fixed slide counts
4. define the image slot rules for AI versus what the layout engine controls
5. rebuild prompts so they inherit a set slot instead of asking Nano Banana to invent the whole card

Immutable by default:

- typography family
- spacing rhythm
- card geometry
- alignment system
- CTA structure
- product fidelity rules

Mutable by default:

- palette temperature inside the approved range
- accent placement
- light softness
- crop
- subject
- background texture
- lifestyle scene choice

## Working Decision

If the team wants one exact answer right now:

Use `8` locked operational sets inside `5` master families.

That is the strongest balance between:

- non-repetition
- brand recognition
- prompt control
- token efficiency
- realistic production overhead
