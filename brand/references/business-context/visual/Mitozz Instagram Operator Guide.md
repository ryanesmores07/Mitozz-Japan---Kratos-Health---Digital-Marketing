# Mitozz Instagram Operator Guide

## Purpose

Use this guide when producing Mitozz Instagram assets.

This is the short operator-facing version of the production system.

## Default Production Order

1. read the calendar row
2. resolve the post-calendar production layer:
   `Template Set`, `Slide Blueprint`, `asset_archetype`, and `story_type` when needed
3. choose the correct approved reference images
4. build the creative package
5. generate the asset or background plate
6. review against the approval rules

## Which Set To Use

First principle:

- default back to `Set 1: Education White Card`
- this is the main Mitozz feed signature
- use the other sets as support modes around it

Educational carousel:

- use `Set 1` and `Set 2`
- finish with `Set 3` only if product context helps
- use the `Set 1` master spacing and full-card structure as the default starting point

Ingredient post:

- use `Set 1` and `Set 2`
- use `Set 3` only on the final frame if needed

Product post:

- use `Set 3`
- use the real-photo bottle workflow

Lifestyle post:

- use `Set 4`
- default to no bottle unless the bottle can stay secondary and realistic

Authority or trust post:

- use `Set 5`

Story reinforcement:

- simplify `Set 1`

Poll or question story:

- simplify `Set 4` or `Set 5`
- follow the story operating rules in [Mitozz Instagram Story Strategy.md](/Users/ernieryanesmores/Desktop/Workspace/Mitozz-Japan---Kratos-Health---Digital-Marketing/brand/references/business-context/visual/Mitozz%20Instagram%20Story%20Strategy.md)

## How The Feed Should Feel

When someone lands on the grid:

- `Set 1` should be the most recognizable and repeated visual language
- `Set 2` should feel invisible as support inside `Set 1`
- `Set 3` should appear as premium product punctuation, not as the dominant mode
- `Set 4` should make the feed feel real and human
- `Set 5` should make the brand feel trustworthy and calm

If the mix ever feels too scattered, increase `Set 1` usage first.

## Typography Rule

Lock typography before approving any asset set:

- cover headline = `Noto Sans JP Bold`
- cover supporting line = `Noto Sans JP Medium`
- inner carousel slides = `Noto Sans JP Medium`
- CTA slides = `Noto Sans JP Medium`
- story frames = `Noto Sans JP Medium`

Do not approve:

- mixed font families across one asset set
- inconsistent font weights between similar slide roles
- fake button styling
- random changes in hierarchy from slide to slide

For `Set 1`, follow:

- [Mitozz Set 1 Master Template Spec.md](/Users/ernieryanesmores/Desktop/Workspace/Mitozz-Japan---Kratos-Health---Digital-Marketing/brand/references/business-context/visual/Mitozz%20Set%201%20Master%20Template%20Spec.md)

## Aspect Ratio Rule

Lock ratios before approving:

- normal feed posts and carousel slides = `4:5`
- square feed posts only when intentionally planned = `1:1`
- stories = `9:16`
- reel source frames = `9:16`

Reject:

- landscape feed outputs
- random ratio drift inside one carousel
- square crops that were not intentionally specified

## Set 1 Layout Rule

When producing `Set 1`:

- the canvas should behave like the rounded white card itself
- use only a minimal pale blue edge atmosphere
- do not use a thick outer gradient frame
- keep text spacing and margins stable across the carousel
- let support visuals stay subtle and internal to the card
- make sure the first slide creates immediate relevance, not only a clean topic label

## Batch Lock Rule

For every connected batch, lock these before approval:

- font family
- font weight by role
- text color
- text opacity
- support-visual opacity
- glow intensity
- overall contrast

Use one visual calibration across the batch.

Reject:

- one frame looking bolder or lighter for no reason
- one slide using stronger glow or darker text than the others
- one asset drifting into a different color temperature
- visible inconsistency in softness or opacity treatment

## Continuous Improvement Rule

Do not repeat a known mistake.

When a generation issue teaches a reusable lesson:

- add it to [Mitozz Generation Lessons Log.md](/Users/ernieryanesmores/Desktop/Workspace/Mitozz-Japan---Kratos-Health---Digital-Marketing/brand/references/business-context/visual/Mitozz%20Generation%20Lessons%20Log.md)
- update the prompt wording if the fix belongs in generation
- update the master spec if the fix should become permanent

Typical lessons worth recording:

- border drift
- rogue CTA text
- typography inconsistency
- export-size mismatch
- over-copying an older asset instead of only preserving the frame language

## Set 3 Product Rule

For every product-led asset:

- use the real bottle photo as product truth
- do not approve AI-redrawn bottles
- use approved background plates for rotation
- composite the bottle into the plate in design

## What Not To Use

Do not use these as reference assets going forward:

- AI-redrawn bottle outputs that changed package truth
- cluttered or object-heavy background plates
- exploratory drafts that are not listed in the approved assets file

## Final Rule

When in doubt:

- preserve the system
- preserve the bottle truth
- preserve the calm premium feel
- let Stories be lighter and more interactive than the feed without breaking the brand

Do not improvise a new visual world unless the system is being formally refreshed.
