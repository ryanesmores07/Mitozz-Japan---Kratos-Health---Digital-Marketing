# Mitozz Story Posting Actions

## Purpose

Use this file at posting time.

This is the operator system for what action to attach to each Story so you do not have to remember it manually.

## Core Rule

Every Story should have one clear posting action.

That action can be:

- `feed-route`
- `poll`
- `slider`
- `question-box`
- `link-sticker`
- `save-reminder`
- `reply-prompt`

Use one primary action only.

## Default Action By Story Type

### `feed-reinforcement`

Default action:

- `feed-route`

Posting rule:

- add a link sticker to the matching feed post when possible
- if direct feed linking is not available in your posting flow, use text guidance plus profile/feed navigation

Best CTA language:

- `続きはフィードへ。`
- `本編はフィードで。`

### `daily-context`

Default action:

- `poll` or `slider`

Posting rule:

- use this when the goal is empathy and self-recognition
- the interaction should feel light and easy, not like a survey

Best CTA language:

- `こんな日、ありますか？`
- `今日はどのくらい余白がありますか？`

### `interactive`

Default action:

- `poll`
- `question-box`
- `slider`

Posting rule:

- choose the sticker that matches the question format
- do not add a second sticker unless the Story is intentionally built for it

### `proof-trust`

Default action:

- `save-reminder` or `reply-prompt`

Posting rule:

- use this when the goal is reflection, reassurance, or quiet trust-building
- keep the interaction softer than a poll-led Story

Best CTA language:

- `参考になったら保存`
- `気になる点があればDMへ`

### `route-cta`

Default action:

- `link-sticker`

Posting rule:

- use when there is a specific destination outside the Story itself
- let the native sticker do the click work

## Sticker Placement Rule

- place the sticker only in the safe space planned in the design
- do not cover the main text block
- keep the Story premium and breathable

## Operator Shortcut

At posting time, decide in this order:

1. identify the `story_type`
2. use the default action for that type
3. apply only one native sticker or one route action
4. do not invent a second CTA unless the Story specifically needs it

## Mitozz Principle

The visual should guide.
The native Story tool should perform the action.

Do not design fake buttons into the artwork.
