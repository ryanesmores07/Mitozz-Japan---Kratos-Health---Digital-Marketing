# Reference Pack Index

This file is the lightweight registry for the official Nano Banana visual reference system.

## Selection Policy

- Reference source: curated pack
- Match strictness: medium
- Each generation should use 2 to 4 image references
- At least 1 reference must be a core `style-anchor`
- Add up to 1 mode-specific `style-anchor` when the asset clearly matches that mode
- Add 1 approved `product-source` reference from `source-intake/` when product fidelity is important
- At most 1 reference may be a close composition match
- If no approved `working-example` exists for the asset type, fall back to `style-anchor` images plus the visual direction docs

## What Must Stay Consistent

- steel-blue and cloud-white atmosphere
- restrained apricot signal accents
- luminous clinical minimalism
- generous whitespace
- calm premium lighting
- editorial restraint in typography and composition

## What Must Rotate

- product crop
- camera distance
- angle
- human vs product balance
- foreground/background treatment
- soft-shape or glow treatment
- layout rhythm between text-first, product-first, and portrait-trust posts

## Feed Planning Rule

No two adjacent assets should use the same dominant composition template unless they are intentionally part of the same campaign sequence.

## Approval Loop For Future References

Only manually approved outputs should be promoted into `working-examples`.

Promote a candidate only if it:

- clearly matches the visual direction
- feels premium and calm
- does not feel repetitive next to the current pack
- improves the brand system instead of diluting it
