# Mitozz Feed Engagement Standard

## Purpose

This file defines how the Mitozz feed should become more engaging without becoming louder, cheaper, or less premium.

The goal is not to redesign the feed.

The goal is to make the feed better at:

- stopping the scroll
- earning the swipe
- increasing saves and shares
- making the brand feel more alive

## Core Principle

Keep the current premium system.

Improve the feed through:

- stronger hook logic
- better slide pacing
- more real-life context
- clearer behavioral CTAs

Do not improve engagement by making the feed noisier.

## What Strong Brands Tend To Do

Successful brands usually do not rely on one post type only.

They consistently combine:

- hook-first opening frames
- educational or narrative pacing
- human or contextual proof
- clear end behavior
- repeated visual systems instead of constant redesign

Mitozz should follow that operating logic inside the existing template system.

## Mitozz Feed Rules

### 1. Hook-First Covers

Every feed carousel should win the first slide quickly.

The cover should do at least one of these jobs:

- create curiosity
- reframe a familiar problem
- make a topic feel immediately relevant

Cover rule:

- do not open with a generic topic label only
- give the topic a reason to matter now

### 2. Narrative Pacing, Not Just Information Pacing

Do not build carousels as a flat list of facts.

Build them as a short narrative:

1. hook
2. reframe
3. explain
4. apply
5. proof or context
6. CTA

For shorter carousels:

- keep the same logic compressed

### 3. One Human Or Contextual Moment

Longer carousels should include at least one slide that feels more lived-in or contextual.

This does not mean every carousel needs a full lifestyle photo.

It means at least one slide should help the viewer feel:

- this applies to real life
- this belongs in a daily routine
- this is not just abstract information

### 4. Rotate CTA Behaviors

Do not end every feed post with the same CTA.

Approved CTA behaviors:

- `save this`
- `send this`
- `read the caption`
- `which one feels familiar?`
- `come back to this later`

The last slide should always guide a behavior, even if it stays visually restrained.

### 5. Use Product Sparingly But Intentionally

Product presence should act like punctuation, not wallpaper.

Use `Set 3` only when:

- product context genuinely helps understanding
- the final frame benefits from a product tie-back
- the post is product-led by strategy

### 6. Use Human Rhythm Across The Grid

The feed should not feel like a sequence of white cards only.

Keep `Set 1` dominant, but add relief through:

- `Set 4` lifestyle posts
- `Set 5` trust posts
- occasional contextual imagery inside `Set 1` carousels

### 7. Let Reels Handle More Discovery

Reels should stay part of the system because discovery behavior often happens there first.

The feed should:

- educate
- organize
- create save value

Reels should:

- widen reach
- create motion and context
- add emotional entry points

## Set 1 Engagement Upgrade

For `Set 1 + Set 2` carousels, use this as the default structure:

1. Hook cover
2. Why it matters
3. Core explanation
4. Practical interpretation
5. Real-life or brand-context slide
6. CTA

What should improve:

- stronger first-slide relevance
- more applied middle slides
- at least one contextual or proof-adjacent moment

What should stay the same:

- premium restraint
- whitespace
- typography system
- calm color world
- full-card structure

## Approval Check

Before approving a feed asset, ask:

1. does the cover make the topic matter quickly?
2. does the carousel earn the swipe, not just request it?
3. is there at least one contextual or humanizing moment if the post is long enough?
4. does the last slide guide one clear behavior?
5. does the post still feel unmistakably Mitozz?

If the answer to `1`, `2`, or `4` is no, the post may be beautiful but still underperform structurally.
