# Mitozz Story Posting Actions 01

## Window

`2026-03-23` to `2026-03-31`

Use this file at posting time.

This file removes guesswork by telling you the exact native Instagram action to attach to each story date.

## Core Rule

Every story should have one primary posting action only.

Do not stack multiple stickers unless there is a real reason.

Priority order:

1. route to feed when the story is reinforcement
2. use a poll when the story is meant to gather a quick signal
3. use a question box when the story is meant to open replies
4. use a save reminder only when the story is reflective and not interaction-led

## Sticker Placement Rule

- Keep stickers in the lower third unless the design clearly leaves safer room elsewhere.
- Do not cover the main text block.
- Leave at least one finger-width of margin from the bottom edge.
- Match the cleanest native style available inside Instagram.
- Do not add decorative GIFs, arrows, or extra UI unless the story specifically needs it.

## Simple Posting Rule

If the story type is:

- `feed-reinforcement` -> use `Link Sticker` to the matching feed post
- `daily-context` -> use `Poll Sticker`
- `interactive` -> use `Poll Sticker` or `Question Sticker`, never both
- `proof-trust` -> default to no sticker unless routing is needed

## Final Check Before Posting

Before you publish any story:

1. confirm the sticker type from this file
2. confirm it is attached only on the last frame unless the story was designed otherwise
3. confirm the sticker does not cover the main text
4. confirm the action matches the story's actual job
5. do not improvise a new CTA if one is already defined here
