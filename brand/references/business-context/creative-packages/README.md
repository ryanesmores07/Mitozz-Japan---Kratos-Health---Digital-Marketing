# Creative Packages

This folder holds the creative-package and handoff files used between calendar planning, prompt creation, generation, and reel editing.

## Keep These As The Main Working Files

- `creative-package-YYYY-MM-DD.md`: one dated creative package for that publish date
- `reel-creative-package-template.md`: starting template when the row format is `Reel`
- `reel-freelancer-handoff-YYYY-MM-DD.md`: filled reel edit packet once source frames are approved
- `reel-edit-blueprint-YYYY-MM-DD.md`: reel timing, text, and beat plan for the editor
- `post-calendar-layer-YYYY-MM-DD-to-YYYY-MM-DD.md`: lean downstream production handoff for a calendar window
- `production-batch-YYYY-MM-DD-to-YYYY-MM-DD.md`: optional control sheet when we need one batch-level status file
- `story-posting-actions-YYYY-MM-DD-to-YYYY-MM-DD.md`: posting-time sticker and CTA instructions for story windows

## Workflow

Pipeline: content calendar -> creatives director -> prompt engineer -> generation / handoff.

Use this folder like this:

- start from `creative-package-YYYY-MM-DD.md` for normal feed and story work
- use `reel-creative-package-template.md` only when the asset is a reel
- use `workflows/04-freelancer-reel-handoff-template.md` to build reel handoff packets after source frames are approved
- use `workflows/05-image-to-video-prompt-template.md` only when an approved still needs to become a motion source clip

## Lean Maintenance Rule

- keep one dated creative package per live publish date
- keep one dated reel handoff only when that reel is actually moving into edit
- keep batch control files only while the calendar window is still operational
- remove one-off bootstrap notes once their process has been absorbed into the main workflow
