# Mitozz Monthly Content Calendar

Use this skill to create or revise the monthly Instagram calendar so downstream creative and prompt-generation work can proceed with minimal reinterpretation.

## Quick Start

1. Identify the requested month, scope, and whether this is a new plan or a revision.
2. Read only the most relevant audience, strategy, brand voice, visual direction, and latest calendar inputs.
3. Use the latest approved calendar as the operating baseline unless the user asks for a strategic reset.
4. Plan the month as one coherent system, not isolated post ideas.
5. Output a calendar that is easy for `mitozz-creatives-director` to consume directly.

## Strategic Role

Build an education-first monthly plan that:

- grows awareness and repeated familiarity before hard conversion
- translates mitochondria-first wellness into simple, relevant Instagram topics
- balances brand building, trust building, and soft conversion support
- reflects Mitozz Japan as premium, calm, science-led, trustworthy, and medically responsible
- creates enough specificity that creative direction can be generated without rethinking strategy

## Required Brand Truths

Treat these as fixed unless a newer brand document overrides them:

- Mitozz Japan is a premium, science-led, mitochondria-first wellness brand built around epicatechin.
- Core themes are daily condition, energy support, quality of life, healthy aging, trust-first education, and better function over hype.
- Instagram is primarily an organic education and trust channel, not a hard-sell channel.
- The account should feel visually disciplined, message-clear, calm, and premium rather than trend-led or noisy.
- Messaging must remain compliant for Japan general-food-safe communication.
- Avoid direct medical, anti-aging, fatigue-recovery, body-transformation, or guaranteed functional claims.
- Prefer education, support language, routines, context, and trust cues.

## Core Personas

Prioritize one primary persona per asset:

1. `Sleep-Deprived High Performer`
2. `Healthy Aging Planner`
3. `Research-First Wellness Optimizer`

Do not blend personas unless a post clearly serves more than one and that helps execution.

## Approved Content Pillars

Use these pillars as the default taxonomy:

1. `Mitochondria, Explained Simply`
2. `Epicatechin and the Science Story`
3. `Better Function, Not Hype`
4. `Trust, Safety, and Premium Proof`

## Planning Workflow

1. Determine the monthly objective:
   - awareness foundation
   - education depth
   - trust building
   - soft conversion support
   - mixed month
2. Set cadence using the existing operating pattern unless a change is justified.
3. Build the month around a clean sequence:
   - foundational education
   - relevance and routine framing
   - ingredient and science translation
   - trust and proof
   - light conversion support when appropriate
4. Pair feed and story content intentionally:
   - reinforcement
   - FAQ
   - objection handling
   - poll
   - question box
   - premium proof
   - education mini-slide
5. Make each row specific enough for creative production:
   - one clear topic
   - one primary persona
   - one core angle
   - one objective
   - one brand-safe CTA
6. Review the month as a system before saving.

## Format Decision Rules

Choose formats deliberately instead of rotating mechanically:

- `Carousel`: best for structured education, save/share value, and stepwise explanation
- `Reel`: best for quick relevance, everyday-condition framing, and simple myth-to-clarity storytelling
- `Static Post`: use sparingly for premium proof, trust moments, or a single strong message
- `Story`: use as support, reinforcement, lighter education, or audience interaction

Favor clarity and coherence over novelty.

## Output Format

Default to a CSV-compatible table with these columns:

- `Publish Date`
- `Day`
- `Format`
- `Working Title / Topic`
- `Content Pillar`
- `Objective`
- `Primary Persona`
- `Hook / Angle`
- `CTA`
- `Caption Status`
- `Creative Status`
- `Asset Link`
- `Notes`

For the story section, keep the current story table structure unless the user requests a new schema.

## Row Quality Standard

Every feed row should make the downstream brief obvious. Make sure each one includes:

- a topic that is specific, useful, and not generic
- a clear educational or trust-building purpose
- a single primary persona
- an angle that can become a strong first-frame message
- a CTA that fits the funnel stage
- notes when the creative system should emphasize a certain visual or relationship to stories

Every story row should include:

- story type
- purpose
- relationship to a feed post when relevant
- simple interaction or CTA when useful
- language light enough for stories but still premium

## CTA Rules

Default to soft CTAs such as:

- `Learn how it works`
- `Explore the mitochondria-first approach`
- `Discover why mitochondrial health matters`
- `See if Mitozz fits your routine`
- `Read more`

Use direct shopping CTAs only when the content is clearly conversion-supportive.

## Decision Rules

- Do not fill the month with random variety for its own sake.
- Do not over-index on product promotion in awareness content.
- Do not use generic supplement-brand angles, fear hooks, or sensational promises.
- Do not rely on empty trend formats if they weaken premium positioning.
- Make the calendar feel like one coherent Mitozz system.
- Prefer clear, execution-ready topics over clever but vague ideas.

## Final Check Before Saving

Confirm the calendar:

- reflects the four approved pillars and three core personas with intention
- supports trust before selling
- stays within calm, premium, science-led brand behavior
- avoids risky claims or hype language
- gives downstream creative work enough direction without rewriting strategy
